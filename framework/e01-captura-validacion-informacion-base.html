4. Reglas e incertidumbres iniciales

Agrimensura

Obligatoria No se puede iniciar Concept sin agrimensura.

Qué reduce: incertidumbre sobre la realidad física del lote.

Debe permitir conocer: límites, geometría, ángulos, niveles de vereda, desniveles y elementos que condicionen accesos o gestiones, como árboles u otros obstáculos relevantes.

Estudio de suelos

Obligatorio Se definió como necesario para Concept.

Qué reduce: incertidumbre sobre estrategia preliminar de fundación y posible impacto económico.

Excepción: prueba de venta contra oferta de compra del lote.

Documentación preexistente

Pendiente Eugenia

No se refiere a factibilidades. Se refiere a planchetas, planos municipales previos u otros antecedentes documentales del lote.

Ficha / input del desarrollador

A definir en etapas siguientes

Debe aportar información inicial para comenzar a estructurar el negocio: unidades, m² vendibles, objetivos comerciales y restricciones iniciales.

5. Gate G01 · Aptitud para iniciar Concept

Objetivo del gate

Determinar si existe información suficiente para iniciar Concept con un nivel de incertidumbre aceptable.

No valida

No valida si el edificio entra en el lote, porque todavía no existe edificio. No valida implantación, tipologías, volumetría ni superficies finales.

Sí valida
  • Que se conoce la realidad física básica del lote.
  • Que existe una base geotécnica suficiente para no iniciar con incertidumbre estructural crítica.
  • Que se cuenta con antecedentes documentales disponibles o se conoce su ausencia.
  • Que el negocio cuenta con información inicial suficiente para comenzar a estructurarse.
Salida recomendada

Base de Concept Validada o Input Package V1.

Debe contener estado de agrimensura, estudio de suelos, documentación preexistente, ficha/input del desarrollador, faltantes, riesgos y observaciones.

ENTREVISTA A EUGENIA · BLOQUE 01 · CAPTURA Y VALIDACIÓN DE INFORMACIÓN BASE

Objetivo:
Validar los criterios reales de la etapa previa a Concept, especialmente documentación preexistente, validación de información base y reglas de liberación.

1. DOCUMENTACIÓN PREEXISTENTE

1.1. Cuando el flujo menciona "documentación preexistente", ¿qué documentos entran exactamente?
Ejemplos a confirmar:
- Planchetas.
- Planos municipales previos.
- Expedientes anteriores.
- Antecedentes del lote.
- Otros.

1.2. ¿Qué documentación preexistente aporta valor real para iniciar Concept?

1.3. ¿Qué documentación suele recibirse pero rara vez se usa?

1.4. ¿Qué documentación preexistente considerás crítica?

1.5. ¿Qué problemas suelen aparecer cuando esa documentación falta o está desactualizada?

1.6. ¿Cómo determinás si un plano municipal previo o plancheta sigue siendo útil?

2. VALIDACIÓN DE INFORMACIÓN BASE

2.1. Cuando el flujo dice "validación de información base", ¿qué validás realmente?

2.2. ¿Validás solo completitud documental o también evaluás riesgos?

2.3. ¿Qué condiciones mínimas deben cumplirse para liberar el inicio de Concept?

2.4. ¿Qué faltantes bloquean completamente el avance?

2.5. ¿Qué faltantes permiten avanzar con observación o riesgo asumido?

2.6. ¿Existe algún checklist actual, aunque sea informal?

2.7. ¿Dónde queda registrada la validación?

3. AGRIMENSURA

3.1. ¿Qué revisás normalmente cuando recibís la agrimensura?

3.2. ¿Detectás errores técnicos o solo verificás existencia y correspondencia del documento?

3.3. ¿Qué defectos harían que pidas corrección de la agrimensura?

3.4. ¿Qué información de la agrimensura es más importante para iniciar Concept?

4. ESTUDIO DE SUELOS

4.1. ¿Qué revisás normalmente cuando recibís el estudio de suelos?

4.2. ¿Derivás la revisión técnica a Antonio o Nahuel cuando aparecen recomendaciones estructurales relevantes?

4.3. ¿Qué tipo de anomalías o conclusiones del estudio suelen generar alerta?

4.4. ¿Cómo se registra que el estudio fue revisado o aceptado?

5. KICK OFF

5.1. ¿El kick off debería hacerse antes de solicitar la documentación o después de recibirla?

5.2. ¿Quién debería convocarlo?

5.3. ¿Quiénes deberían participar?

5.4. ¿Qué resultado concreto debería dejar el kick off?

6. DECISIÓN DE LIBERACIÓN

6.1. Cuando liberás un proyecto para iniciar Concept, ¿qué estás diciendo exactamente?

6.2. ¿La salida debería llamarse "Base de Concept Validada", "Input Package V1" u otro nombre?

6.3. ¿A quién se informa que el proyecto queda liberado?

6.4. Si el proyecto no queda liberado, ¿cómo se comunica el rechazo o pedido de corrección a la oficina del desarrollador?
